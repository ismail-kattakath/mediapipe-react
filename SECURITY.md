# Security Policy

## Reporting a Vulnerability

We take the security of our project seriously. If you believe you have found a security vulnerability in this project, please report it to us as described below.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please send an email to [ismail.kattakath@gmail.com](mailto:ismail.kattakath@gmail.com).

Please include the following information in your report:

- Type of issue (e.g., buffer overflow, SQL injection, cross-site scripting, etc.)
- Full path of source file(s) related to the manifestation of the issue
- The location of the affected potentially vulnerable code (line reference)
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit the issue

We will acknowledge receipt of your report within 48 hours and provide a timeline for addressing the issue.
